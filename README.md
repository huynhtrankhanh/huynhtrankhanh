I used to be working on Contrasleuth, a project that is conjectured to be able to create a mesh network of mobile Android devices. My conjecture has been proven wrong because the discoverServices API is heavily rate limited (my intention was to abuse that API to transmit QUIC packets), the Wi-Fi Direct API requires user interaction and Bluetooth is way too slow as its intended use case is to transmit small amounts of data without draining battery. ~~**New information available:** [the Nearby framework](https://developers.google.com/nearby/connections/overview) is a pretty advanced builtin framework that can work around the limitations of wireless technologies! **I will consider reviving Contrasleuth.**~~ _Pigs can't fly. [The Nearby framework uses Bluetooth and Bluetooth is useless for Contrasleuth.](https://stackoverflow.com/a/54470958) Rest in peace, Contrasleuth. Android can't form mesh networks. It's a platform limitation. And there is no way to circumvent this._

I have been infatuated with formal verification since I was in grade 8 and I think now it's time to act on that desire.
